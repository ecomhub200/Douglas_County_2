### 1.6 Warrant Calculation Updates

**Current calculation** (`stopsign_updateVolumeAnalysis()` at line 39811):
- Reads `stopsignMajor_${hour}` and `stopsignMinor_${hour}` directly

**New calculation**:
- Calculate Major Street = EB_total + WB_total (if major is EW)
- Calculate Minor Street = NB_total + SB_total (if major is EW)
- Compare against thresholds (300/200 or 210/140 if 70% applied)

```javascript
function stopsign_calculateApproachVolumes(hour) {
  const majorDir = warrantsState.stopsign.config.majorDirection;
  const isMajorEW = majorDir === 'EW';

  const approaches = ['NB', 'SB', 'EB', 'WB'];
  let majorSum = 0, minorSum = 0;

  for (const app of approaches) {
    const total = parseInt(document.getElementById(`stopsign_tmc_${hour}_${app}_total`)?.value) || 0;
    const isMajor = (isMajorEW && (app === 'EB' || app === 'WB')) ||
                    (!isMajorEW && (app === 'NB' || app === 'SB'));
    if (isMajor) majorSum += total;
    else minorSum += total;
  }

  return { major: majorSum, minor: minorSum };
}
```

### 2.3 AI Extraction Prompt (Stop Sign Specific)

**Key differences from Signal Warrant extraction**:

1. **Intersection type awareness**:
   - Identify if it's 2-way stop, 4-way stop, or yield
   - Note which approaches are stop-controlled

2. **8-hour analysis focus**:
   - MUTCD requires 8 highest hours meeting threshold
   - Extract full day but highlight relevant 8 hours

3. **Simpler configuration**:
   - No U-turn columns needed typically
   - Speed threshold affects volume requirements

**Extraction prompt template**:
```javascript
const STOPSIGN_EXTRACTION_PROMPT = `
You are a traffic data extraction expert. Extract turning movement count (TMC) data from this traffic count file.

EXTRACTION REQUIREMENTS:

1. INTERSECTION CONFIGURATION:
   - Intersection name (if visible)
   - Major street name
   - Minor street name
   - Number of legs (3 or 4)
   - Any posted speed limit visible
   - Existing control type if mentioned

2. DATE & DAY INFORMATION:
   - Look for count date in file header, footer, or filename
   - Determine day of week
   - Format: YYYY-MM-DD

3. TMC DATA STRUCTURE:
   - Identify direction columns: NB, SB, EB, WB (or N, S, E, W)
   - For each direction, extract: Left (L), Through (T), Right (R)
   - Total should be L + T + R
   - Extract ALL hours with data

4. RETURN FORMAT (JSON):
{
  "intersection": "Main St & Oak Ave",
  "majorStreet": "Main St",
  "minorStreet": "Oak Ave",
  "intersectionLegs": 4,
  "speedLimit": null,
  "date": "2025-01-15",
  "dayOfWeek": "Wednesday",
  "confidence": 95,
  "hourlyVolumes": {
    "6": {
      "NB": { "left": 5, "thru": 45, "right": 10, "total": 60 },
      "SB": { "left": 8, "thru": 38, "right": 12, "total": 58 },
      "EB": { "left": 20, "thru": 150, "right": 25, "total": 195 },
      "WB": { "left": 15, "thru": 140, "right": 20, "total": 175 }
    }
  },
  "grandTotalInFile": 19500,
  "extractedTotal": 19450
}

IMPORTANT:
- Extract values EXACTLY as shown in the file
- If a cell is empty or blank, use 0
- Calculate totals as L + T + R for verification
- Note any data quality issues in a "warnings" array
`;
```

```javascript
// 1. User uploads files

  └── Updates slot indicators (D1-D5)
  └── Enables "Extract & Validate" button

  └── For each file:
      └── stopsign_extractSingleFileWithDualAI(file, slot)

          └── API call to Claude (validation prompt)
          └── Returns { extractedData, validationResults }
  └── stopsign_agent3ReExtract(allExtractions)
      └── Cross-file variance check
      └── Re-extract anomalies if needed
  └── stopsign_generateDataPreview(finalExtractions)
      └── Render preview panel

// 3. User confirms extractions
stopsign_confirmExtractedData()
  └── stopsign_enterReviewMode()
      └── Load first extraction into TMC table
      └── stopsign_populateTMCGridFromExtraction(ext)

// 4. User reviews each day
stopsign_addCurrentDay()  // For each day
  └── Collect TMC values from table
  └── Store in warrantsState.stopsign.multiDayData
  └── stopsign_renderDayCards()
  └── Load next extraction if in review mode

// 5. Warrant evaluation updates automatically
stopsign_evaluateAllCriteria()
  └── stopsign_calculateApproachVolumes() for each hour
  └── Update C.1, C.2 status
  └── Update overall warrant result
```
