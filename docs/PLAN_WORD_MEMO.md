# Word Memo Export Feature - Implementation Plan

## Overview

Add a Word Memo export capability to the Reports tab, allowing users to generate professional memorandum documents for crash analysis reports. This follows the established pattern from the Signal Warrant Word Memo (`signal_generateWordMemo()`).

## Goal

Create a "Word Memo" export option that generates professional `.docx` memorandum documents for:
- Corridor/Segment Analysis
- Intersection Analysis
- System-Wide Summary
- Safety Performance Report

## Architecture Decision

### Approach: Extend Existing Reports Tab

**Why this approach:**
1. Reuses existing report generation logic (`computeStats`, `generateFindings`, `generateRecommendations`)
2. Consistent with user workflow (same tab, same location/date selection)
3. Minimal new UI - just add a "Download Word" button next to existing "Download PDF"
4. Follows established DOCX.js patterns already in codebase

**Alternative considered:** Standalone export section like Signal Warrant
- Rejected because: Would duplicate location/date selection UI and create fragmented user experience

### Step 2: Create Main Word Memo Generator Function

**New function:** `downloadReportWord()`

```javascript
async function downloadReportWord() {
    const type = document.getElementById('reportType').value;
    const title = document.getElementById('reportTitle').value || 'Crash Analysis Report';
    const author = document.getElementById('reportAuthor').value || 'Traffic Engineering Division';

    // Route based on report type
    switch(type) {
        case 'systemwide':
            await generateSystemwideWordMemo(title, author);
            break;
        case 'corridor':
        case 'intersection':
            await generateCorridorWordMemo(title, author);
            break;
        case 'safety':
            await generateSafetyWordMemo(title, author);
            break;
        default:
            await generateGenericWordMemo(title, author, type);
    }
}
```

#### 3.2: `generateCorridorWordMemo(title, author)`

Similar structure but focused on single corridor/intersection:
- Location-specific statistics
- Node/segment breakdown if corridor
- Crash clustering analysis

#### 3.3: `generateSafetyWordMemo(title, author)`

Focus on K+A crashes:
- Severe crash patterns
- Lighting condition analysis
- Contributing factors
- Targeted safety recommendations

## Data Flow

```
User clicks "Download Word"
        │
        ▼
downloadReportWord()
        │
        ├─► Get report type, title, author from UI
        │
        ▼
Route to specific generator
        │
        ├─► Get crashes (same filter logic as PDF reports)
        │   └─► crashState.sampleRows + date/route filters
        │
        ├─► computeStats(crashes)
        │
        ├─► generateFindings(stats, crashes)
        │
        ├─► generateRecommendations(stats, crashes)
        │
        ▼
Build DOCX document
        │
        ├─► buildMemoHeader()
        ├─► buildStatsTable()
        ├─► buildFindingsSection()
        ├─► buildLocationTable()
        ├─► buildRecommendationsSection()
        │
        ▼
docx.Packer.toBlob(doc)
        │
        ▼
Trigger download
```

## File Naming Convention

```
{ReportType}_Memo_{Location}_{YYYY-MM-DD}.docx

Examples:
- Systemwide_Memo_Henrico_County_2025-01-15.docx
- Corridor_Memo_Broad_Street_2025-01-15.docx
- Safety_Memo_Intersection_Analysis_2025-01-15.docx
```
