Phase 1 — External Data Sources
Multiple sources fetched in parallel
SOURCE State DOT Crash APIs
50+ state data portals — ArcGIS REST APIs, Open Data portals, CSV downloads. Virginia uses the Virginia Roads ArcGIS FeatureServer with 2,000-record pagination.
click to expand
VA API services.arcgis.com/.../CrashData_Basic_Updated
Fallback virginiaroads.org/api/download/v1/... (CSV)
CO Source CDOT Open Data Portal (ArcGIS)
MD Source Maryland ACRS crash reporting system
Pagination 2,000 records/request, auto-paginated
Retry Exponential backoff: 2s → 4s → 8s → 16s (max 4)
Output {jurisdiction}_all_roads.csv

Phase 3 — Unified Processing Pipeline
.github/workflows/pipeline.yml → scripts/*.py
STAGE 0 Initialize Cache
Creates state-isolated cache directories for incremental processing. Previously validated rows and geocoded coordinates are cached to skip on re-runs.
click to expand
Script scripts/init_cache.py
Output .cache/{state}/validation/, .cache/{state}/geocode/
STAGE 1 Validate
Runs quality checks on the full statewide dataset: required fields, date ranges (1995–2030), KABCO severity codes, coordinate bounds with 2% GPS drift tolerance, and duplicate detection.
click to expand
Script scripts/validate_data.py
Checks Required fields, date format/range, severity codes, coordinate bounds, duplicates, transposed coords
Bounds State-specific from states/{state}/config.json + 2% buffer
Cache .cache/{state}/validation/ — incremental, hash-based
Log .validation/corrections_log.csv
Skippable Yes — skip_validation: true
STAGE 2 Geocode
Fills in missing GPS coordinates using a fallback chain of free geocoding services and intersection node/offset data. Outputs a gzipped copy for statewide datasets.
click to expand
Script scripts/geocode_data.py
Methods Multiple geocoding APIs (fallback chain) + intersection nodes
Cache .cache/{state}/geocode/
Output {state}_statewide_validated_geocoded.csv + .csv.gz
Timeout 60 minutes max
STAGE 3 Split by Jurisdiction
Splits the single statewide CSV into per-jurisdiction files. Virginia → 133 county CSVs. Colorado → 64 county CSVs. Only runs in statewide/batch mode.
click to expand
Script scripts/split_jurisdictions.py
Input Statewide validated+geocoded CSV
Condition Only when download_mode == 'statewide'
Config config.json jurisdiction definitions + states/{state}/hierarchy.json
Output data/{state}/{jurisdiction}_all_roads.csv × N
STAGE 4 Split by Road Type
Creates 3 road-type variants per jurisdiction: county roads only, no-interstate, and all roads. Virginia: 133 × 3 = 399 CSV files.
click to expand
Script scripts/split_road_type.py
Variants county_roads.csv — local roads only
no_interstate.csv — excludes interstates
all_roads.csv — complete dataset
Filter SYSTEM column + route exclusion patterns
STAGE 5 Aggregate by Scope
Generates scope-aware aggregate CSVs — by jurisdiction, region, MPO, or statewide. Also produces federal cross-state aggregates for statewide runs.
click to expand
Script scripts/aggregate_by_scope.py
Scopes jurisdiction, region, MPO, statewide, federal
Region data/{state}/_region/{name}/
MPO data/{state}/_mpo/{name}/
Federal data/_federal/ — cross-state (statewide only)
STAGE 6 Upload to Cloudflare R2
Uploads all processed CSVs to the crash-lens-data R2 bucket for CDN delivery. Includes county CSVs, region/MPO aggregates, federal aggregates, and statewide gzip.
click to expand
Script scripts/upload-to-r2.py (+ inline AWS CLI in workflow)
Bucket crash-lens-data
CDN URL https://data.aicreatesai.com/{state}/{jurisdiction}/
Sub-steps 6a: County road-type CSVs
6b: Region/MPO aggregate CSVs
6c: Federal aggregates (statewide only)
6d: Statewide gzip (.csv.gz)
Manifest data/r2-manifest.json — tracks sizes, hashes, timestamps
STAGE 7 Generate Forecasts (ML)
Runs ML models (Chronos, SageForecasting) on per-jurisdiction road-type CSVs to predict future crash rates by severity. Forecast JSONs uploaded to R2 (Stage 7b).
click to expand
Script scripts/generate_forecast.py
Models Chronos, SageForecasting (AWS SageMaker)
Input Per-jurisdiction road-type CSVs (3 per county)
Output forecasts_{road_type}.json per jurisdiction
Upload Stage 7b → R2 as {state}/{jurisdiction}/forecasts_*.json
Timeout 60 minutes max

Commits the R2 manifest, validation logs, and split reports back to the git repo. Pushes to main with retry + rebase on conflict.

APP Browser Application — app/index.html
Single-page application loads crash data via PapaParse streaming CSV parser. User selects jurisdiction → fetches from R2 → parses in 1,000-row chunks → populates global state objects → renders across all tabs.
