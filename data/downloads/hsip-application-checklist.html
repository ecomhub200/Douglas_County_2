HSIP Application Checklist

Complete Guide for Highway Safety Improvement Program Grant Applications

Provided by CRASH LENS - Professional Crash Analysis Tools

Introduction

The Highway Safety Improvement Program (HSIP) is a core Federal-aid program designed to achieve significant reductions in traffic fatalities and serious injuries on all public roads. This checklist will guide you through preparing a competitive HSIP application with proper crash data documentation.

Pro Tip: Start your application preparation at least 3 months before the deadline to allow adequate time for crash data analysis and stakeholder coordination.

Phase 1: Project Identification & Eligibility

  • Verify road ownership and functional classification Confirm the project location is on a public road eligible for HSIP funding. Check VDOT's road inventory system.
  • Document crash history (minimum 3-5 years) Obtain official crash records from Virginia DMV or TREDS database. Include all reportable crashes.
  • Calculate crash rates and compare to statewide averages Use AADT data to calculate crashes per million vehicle miles traveled (MVMT) or per million entering vehicles (MEV).
  • Identify crash patterns and contributing factors Analyze crash types (angle, rear-end, head-on), severity levels, and common contributing factors.
  • Confirm project aligns with Virginia SHSP emphasis areas Reference Virginia's Strategic Highway Safety Plan for priority emphasis areas and proven countermeasures.

Phase 2: Crash Data Analysis Requirements

  • Compile crash data summary table Include year-by-year breakdown of total crashes, injuries, and fatalities for the study period.
  • Create collision diagram Show crash locations, types, and movements on a scaled diagram of the project area.
  • Document crash severity distribution Use KABCO scale: K (Fatal), A (Incapacitating), B (Non-incapacitating), C (Possible injury), O (Property damage only).
  • Analyze temporal patterns Identify peak crash times, day-of-week patterns, and seasonal variations.
  • Calculate equivalent property damage only (EPDO) score Weight crashes by severity to demonstrate comprehensive safety impact.

Phase 3: Countermeasure Selection

  • Select FHWA Proven Safety Countermeasures when applicable Applications using proven countermeasures receive priority consideration and streamlined review.
  • Document expected Crash Reduction Factor (CRF) Use CMF Clearinghouse values or Virginia-specific CRFs where available.
  • Ensure countermeasures address identified crash patterns Match proposed improvements to documented crash types and contributing factors.
  • Consider systemic safety approach for widespread patterns Systemic projects addressing multiple locations may be more competitive than single-site projects.
FHWA Proven Safety Countermeasures (2023): Roundabouts, Road Diets, Pedestrian Hybrid Beacons, Leading Pedestrian Intervals, Backplates with Retroreflective Borders, Rumble Strips, Safety Edge, Enhanced Delineation for Curves.

Phase 4: Benefit-Cost Analysis

  • Calculate expected crash reduction benefits Apply CRF to current crash experience to estimate annual crashes prevented.
  • Apply VDOT crash cost values Use current Virginia-approved crash cost values for each severity level.
  • Estimate project costs (construction + maintenance) Include engineering, right-of-way, construction, and 20-year maintenance costs.
  • Calculate Benefit-Cost ratio Target B/C ratio of 1.5 or higher for competitive applications.
  • Document assumptions and methodology Clearly explain how benefits and costs were calculated for reviewer verification.

Phase 5: Application Documentation

  • Complete HSIP application form Download current version from VDOT's Local Assistance Division website.
  • Attach crash data summary and analysis Include tables, charts, and collision diagrams supporting the safety need.
  • Include location maps and aerial photographs Clearly show project limits, crash locations, and proposed improvement areas.
  • Provide preliminary cost estimate Break down costs by major work items with reasonable unit prices.
  • Document local match commitment HSIP typically requires 10% local match. Include resolution or commitment letter.
  • Obtain stakeholder support letters Letters from local government, law enforcement, or community groups strengthen applications.

Phase 6: Pre-Submission Review

  • Verify all required sections are complete Use VDOT's application checklist to confirm no sections are missing.
  • Review crash data accuracy Double-check crash counts, dates, and severity classifications against source data.
  • Confirm B/C calculation methodology Ensure calculations follow VDOT guidelines and are clearly documented.
  • Obtain required signatures Ensure all authorization signatures are in place before submission.
  • Submit by deadline with confirmation receipt Electronic submissions preferred. Request confirmation of receipt.

Common Application Mistakes to Avoid

  • Insufficient crash history Using less than 3 years of data or failing to include all reportable crashes.
  • Mismatched countermeasures Proposing improvements that don't address the documented crash patterns.
  • Incomplete benefit-cost analysis Missing maintenance costs, using outdated crash values, or unclear methodology.
  • Poor quality maps and diagrams Illegible collision diagrams or maps that don't clearly show project limits.
  • Missing local match documentation Failing to provide evidence of committed local funding share.
