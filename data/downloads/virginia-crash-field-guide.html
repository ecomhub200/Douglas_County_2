1. Introduction to Virginia TREDS

The Traffic Records Electronic Data System (TREDS) is Virginia's comprehensive crash reporting system maintained by the Virginia Department of Motor Vehicles (DMV). All reportable crashes in Virginia are documented in TREDS using standardized data elements that comply with the Model Minimum Uniform Crash Criteria (MMUCC).

Reportable Crash Criteria

In Virginia, a crash must be reported if it results in:

  • Injury or death to any person
  • Total property damage of $1,500 or more
  • Damage to state or local government property
Note: Crashes below the reporting threshold are not captured in TREDS data. When analyzing crash patterns, remember that minor property-damage-only crashes may be underrepresented.

2. Crash-Level Fields

These fields describe the overall crash event and are fundamental to any crash analysis.

Field Name Description Format
CRASH_ID Unique identifier for each crash record Numeric (12 digits)
CRASH_DATE Date the crash occurred YYYY-MM-DD
CRASH_TIME Time the crash occurred (24-hour format) HH:MM
DAY_OF_WEEK Day of week (1=Sunday through 7=Saturday) Numeric (1-7)
TOTAL_VEHICLES Number of motor vehicles involved Numeric
TOTAL_PERSONS Total persons involved (occupants + pedestrians) Numeric
CRASH_SEVERITY Most severe injury in crash (KABCO scale) Character (K/A/B/C/O)

3. Location Fields

Location data enables spatial analysis and crash mapping. Virginia uses multiple location systems.

Field Name Description Format
LATITUDE GPS latitude coordinate Decimal degrees
LONGITUDE GPS longitude coordinate Decimal degrees (negative)
ROUTE_NUMBER State route number (if applicable) Alphanumeric
ROUTE_SUFFIX Route direction/type (NB, SB, ALT, etc.) Character
MILEPOINT Linear reference milepoint on route Decimal
COUNTY_CODE FIPS county code Numeric (3 digits)
CITY_CODE City/town code Numeric
INTERSECTION_ID Intersection identifier (if at intersection) Alphanumeric
Data Quality Alert: GPS coordinates may have accuracy issues, especially for crashes reported from memory rather than at the scene. Always verify location using multiple fields when possible.

4. Severity Classification (KABCO Scale)

Virginia uses the KABCO injury severity scale, which is the national standard for crash reporting.

Code Severity Level Definition
K Fatal Death within 30 days of the crash resulting from injuries sustained in the crash
A Suspected Serious Injury Incapacitating injury preventing normal activities (broken bones, severe lacerations)
B Suspected Minor Injury Non-incapacitating, evident injury (bruises, minor cuts, complaint of pain)
C Possible Injury No visible injury but complaint of pain, momentary unconsciousness, or limping
O No Apparent Injury Property damage only, no injuries reported

Equivalent Property Damage Only (EPDO) Weights

EPDO weighting allows comparison of crash severity across locations:

Severity Typical EPDO Weight Virginia HSIP Weight
Fatal (K) 500 1,000
Serious Injury (A) 100 100
Minor Injury (B) 10 10
Possible Injury (C) 5 5
PDO (O) 1 1

11. Analysis Best Practices

Data Preparation

  • Always verify crash counts against official TREDS reports
  • Exclude crashes coded as "Non-Reportable" or "Supplemental"
  • Use consistent study periods (typically 3-5 years of data)
  • Account for year-to-year AADT changes when calculating rates

Common Analysis Types

  • Crash Rate Analysis: Crashes per million vehicle miles traveled (MVMT)
  • Intersection Analysis: Crashes per million entering vehicles (MEV)
  • Pattern Analysis: Identify predominant crash types and contributing factors
  • Temporal Analysis: Time-of-day and seasonal patterns
  • Severity Analysis: EPDO scores and fatal/injury proportions
Formula Reference:
Crash Rate (per MVMT) = (Crashes × 1,000,000) / (AADT × 365 × Years × Length in miles)

Intersection Rate (per MEV) = (Crashes × 1,000,000) / (Entering Vehicles × 365 × Years)

12. Data Quality Considerations

Known Data Limitations:
  • GPS coordinates may be inaccurate (verify with milepoint or intersection data)
  • Contributing factors are officer judgment and may not reflect true causation
  • Minor PDO crashes may be underreported
  • Severity can be upgraded if victims die within 30 days (data may change)
  • Historical data may use different coding systems

Quality Assurance Checks

  • Cross-reference GPS coordinates with described location
  • Verify crash counts match official summaries
  • Check for duplicate crash records
  • Validate severity against person-level injury data
  • Confirm roadway characteristics match known conditions
